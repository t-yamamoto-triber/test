前提：ツールが扱う広告運用の仕組み

MEDIA TOOL

📱 スマートニュース

広告を配信する媒体の管理画面。配信コスト・インプレッション・クリック数などの広告配信データが日別で確認・ダウンロードできる。

MEASUREMENT TOOL

📐 AFAD（計測ツール）

実際にユーザーが商品を購入・申し込んだ「獲得（CV）」を計測するツール。媒体データと突き合わせることで、実際の広告効果を把握できる。

💡 マージン管理について

広告運用では、日々の新規獲得効率（CPA）が目標より高い場合に、弊社のマージンを低く設定して調整することがあります。 月間の目標獲得単価に向けて全体をコントロールするため、日々マージンを管理しています（基本的には同じマージンで進行）。

そのため従来は、毎日の結果を確認してマージンの調整作業が発生していました。このツールにより、その確認・入力作業を効率化しています。

このツールで何が変わったか

BEFORE ／ 自動化前

毎日こんな作業をしていた

  • スマートニュースにログインして CSV をダウンロード
  • AFAD にログインして CSV をダウンロード
  • それぞれスプレッドシートに貼り付け・集計
  • 日々の CPA を確認してマージンを手動で調整
  • 月間予算の進捗・残余予算を手計算
AFTER ／ 自動化後

担当者がやることは1つだけ

  • 届いた通知のリンクを開く
  • マージンを選んでボタンを押す（30秒）
  • レポート画像が自動で届く ✅
  • ダッシュボードで予算進捗をひと目で確認
  • 計算不要・コピペ不要・手入力不要
週 30〜40分　の削減
日次作業 5〜10分 × 5日 ＋ マージン調整・予算計算 週30分 ≒ 月換算で約3〜4時間の工数削減

使用ツール一覧

🎭 Playwright
ブラウザ自動操作
スマートニュース・AFAD に自動ログインし、日付設定からCSVダウンロードまでの操作を完全自動化。通常では自動化しにくい特殊な画面操作にも対応。
⚡ GitHub Actions
定期実行・クラウド処理
毎日0時にクラウド上で自動起動。データ取得→集計→通知→スクリーンショット送信まで、PCを起動していなくても全工程を実行する処理の司令塔。
📜 Google Apps Script
スプレッドシート連携・API
スプレッドシートの読み書き（マージン書き込み・集計データ取得）を担当。マージン入力フォームからのリクエストを受け取り、スプレッドシートに反映するサーバーとして機能。
📊 Google スプレッドシート
データ集計・管理
スマートニュース・AFADからダウンロードしたCSVの保存先。マージンを掛けた売上・CPA などを自動計算し、ダッシュボードの数値ソースとして機能。
💬 Chatwork API
通知・レポート送信
マージン入力依頼・完了確認・レポート画像の3種類のメッセージを自動送信。既存の業務チャンネルにそのまま届くため、担当者の操作変更不要。
🟣 Slack API
通知・レポート送信
Chatwork と並行して同じ内容を送信。テキスト送信（Bot Token + chat.postMessage）と画像共有（GitHub Release 経由でURL取得後に Block Kit で表示）を実装。
🐙 GitHub Releases API
画像の公開ホスティング
Playwright で撮影したダッシュボードのスクリーンショットをリリースアセットとしてアップロードし、公開URLを取得。Slack に画像として表示させるために利用。
🌊 Surge.sh
静的サイトのホスティング
マージン入力フォーム（input.html）とダッシュボード（dashboard.html）を無料でWeb公開。Chatwork通知に記載したURLはここにリンク。

工夫したポイント・ぶつかった課題

🎯 設計上の工夫

① 通知を3ステップに分けた

「依頼 → 完了確認 → レポート」の順で届くため、担当者が今どの状態かを把握しやすい

② 日付は「前日」を表示

深夜0時台に動くため「前日の日付」を通知に使い、データ対象日のズレを防止

③ ChatworkとSlack、両方に対応

移行期間中でも両方に送れる設計。将来 Slack に完全移行してもコード変更不要

④ 予算進捗をひと目で確認できるダッシュボード

月次予算に対する消化率・残余予算が自動計算され、毎日の手計算が不要に（UIは改善予定）

課題 1
ツールへの自動ログインが難しい
スマートニュースは二段階認証があるため、初回のみ手動ログインし、認証情報を安全に保存して以降は自動復元。
課題 2
AFAD の日付入力が自動化しづらい構造
通常の操作では反応しない特殊な日付入力欄のため、画面のボタンを JavaScript で直接クリックする方式に変更。
課題 3
Slack への画像送信がなかなか動かなかった
Slack の API 不具合と外部ホストのブロックを経て、GitHub 上に一時保存した URL を Slack に送る方式で解決。
課題 4
クラウド実行特有のエラーが連発
ローカルとクラウドの環境差異・コマンドの違い・コード更新タイミングのズレなど、1つずつ対処してノウハウを記録。
